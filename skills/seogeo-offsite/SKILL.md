# seogeo · 站外内容矩阵 + 实体权威层

AI 回答约 91% 来自第三方站点而非品牌官网——纯站内优化不够，必须配高权重第三方平台分发。每家 AI 主要"吃自己生态"，所以**按目标引擎选平台**。

## 先拿确定性矩阵（CLI）
先跑 `seogeo offsite [--engine <豆包|元宝|文心|通义|DeepSeek|Kimi>] [--audience b2b|consumer]`，拿到"平台 → 喂哪些引擎 × 受众 × 开放/封闭"的确定性矩阵 + 一题多发清单，再按下面判断细化打法。

## 引擎 → 该铺的平台（每家吃自己生态）

| 目标引擎 | 主铺平台 |
|---|---|
| 豆包 | 抖音 / 今日头条 / 抖音百科 |
| 腾讯元宝 | 微信公众号 / 视频号 |
| 文心 / 百度AI搜索 | 百度百科 / 百家号 |
| 通义 / 夸克 | 网易号 / 企鹅号 / 搜狐号等门户自媒体 |
| DeepSeek / Kimi | 知乎 / CSDN / 掘金 + 公网媒体 |
| ChatGPT / Perplexity / Claude / Google AI | Wikipedia / Reddit / LinkedIn / 权威媒体 |

> 跨引擎共性：搜狐号权重普遍偏高、知乎高价值。封闭型平台（小红书"点点"只引小红书、抖音 AI 只引抖音）→ 想被引就得做平台内 SEO。

## 打法（判断层）

- **按行业分流**：B2B / 科技 → 公众号 + 知乎 + CSDN（喂 DeepSeek / 文心 / 元宝）；消费 / 生活 → 小红书 + 抖音 + B站（喂 豆包 / 通义）。
- **一题多发**：一篇好内容改写成各平台变体，同步发头条号 / 百家号 / 知乎 / 搜狐号 / CSDN——一次产出、多源覆盖。
- **平台定制变体**：知乎重论证、小红书重场景图文、公众号重深度、CSDN / 掘金重代码与实操——别一稿到处糊。
- **持续性**：AI 联网偏好新内容，按月持续发，别一次性。

## 实体 / 权威层（E-E-A-T，让 AI 认得你是"谁"）

- **百科**：能上百度百科 / Wikipedia / 抖音百科就上——AI 认实体的高权重锚点。
- **sameAs 一致**：官网 Organization schema 的 `sameAs` 把百科、知乎机构号、官方主页串起来，告诉 AI"这些都是同一个我"。
- **NAP 一致**：名称 / 地址 / 电话（Name-Address-Phone）全网写法统一，避免实体被拆成多个。
- **作者背书**：文章挂真实作者 + 履历（Person schema），撑起 E-E-A-T 的"经验 / 专业"。

## 百度系结构化提交（国内特有，别套 Google 那套）

- 百度搜索资源平台：提交 sitemap、用结构化数据工具提报；百家号同步原创。
- 注意：熊掌号 / MIP 已过时，别作重点。

## 诚实边界

- 各平台引用占比是单行业样本，方向可信、数值仅参考。
- 站外见效慢且需持续；AI 联网排名波动大、可被操控 → 配 `seogeo-monitor` 按月追踪，别指望一次到位。
